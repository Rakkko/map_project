# OpenMapTiles + Martin 离线地图平台操作指南

# 1. 整体流程说明

整个项目的工作流程如下：

```text
下载地图数据（OSM）
        ↓
使用 OpenMapTiles 处理数据
        ↓
生成 MBTiles 文件
        ↓
使用 Martin 发布瓦片服务
        ↓
使用 Nginx 提供前端页面
        ↓
MapLibre 显示地图
```

各部分功能分别为：

- OSM = 原始地图数据
- OpenMapTiles = 地图加工
- MBTiles = 地图数据库文件
- Martin = 地图服务器
- Nginx = Web服务器
- MapLibre = 地图显示页面

---

# 2. 项目目录结构

初始化完成后目录如下：

```text
.
├── client
│   ├── docker-compose.yaml
│   ├── docker_image
│   ├── frontend
│   ├── martin
│   └── nginx
│
├── host
│   └── openmaptiles
│
└── Makefile
```

## 目录作用

### client

地图发布环境。

负责：

- Martin 服务
- Nginx 服务
- 前端页面

### host/openmaptiles

地图数据处理环境。

负责：

- 下载地图数据
- 导入 OSM
- 生成 MBTiles

### client/martin

存放最终生成的地图文件。

例如：

```text
client/martin/china.mbtiles
```

# 3. 第一次使用：环境初始化

首次使用必须执行。

该步骤会下载：

- Docker镜像
- OpenMapTiles源码
- 字体资源（Glyphs）

执行：

```bash
make all
```

## 执行过程

系统会自动完成：

```text
下载 Martin 镜像
      ↓
下载 Nginx 镜像
      ↓
保存镜像
      ↓
下载 OpenMapTiles
      ↓
下载字体资源
      ↓
同步前端资源
```

## 如何确认成功

检查目录：

```text
host/openmaptiles
```

已存在 OpenMapTiles 文件。

同时：

```text
client/docker_image
```

中存在 Docker 镜像归档文件。

# 4. 设置地图最大缩放级别

## 什么是 Zoom Level

Zoom Level 表示地图可放大的最大层级。

例如：

```text
10 → 城市级
14 → 街道级
18 → 建筑级
20 → 高精度显示
```

默认值一般为：

```text
14
```

---

## 修改缩放级别

执行：

```bash
make set-max-zoom zoom_level=20
```


---

## 缩放级别越高有什么影响

| 项目 | 影响 |
|--------|--------|
| 生成时间 | 增加 |
| 磁盘占用 | 增加 |
| MBTiles大小 | 增加 |
| 内存占用 | 增加 |

# 5. 在线下载地图并生成 MBTiles

这是最常用的方法。

适用于：

没有现成 PBF 文件。

---

## 下载中国地图

执行：

```bash
make download-data area=china
```

---

## 下载四川地图

执行：

```bash
make download-data area=asia/china/sichuan
```

---

## 指定最大缩放级别

例如：

```bash
make download-data area=china zoom_level=12
```

---

## 自动执行内容

系统将自动完成：

```text
下载PBF
    ↓
导入OSM数据
    ↓
导入Wikidata
    ↓
生成数据库
    ↓
生成矢量瓦片
    ↓
输出MBTiles
```

---

## 生成结果

最终文件：

```text
client/martin/area.mbtiles
```

---

# 6. 使用已有 PBF 文件生成地图

如果已经下载了地图数据，可以直接生成。

---

## 第一步：复制 PBF 文件

例如：

```text
china.osm.pbf
```

放到：

```text
host/openmaptiles/data/
```

最终路径：

```text
host/openmaptiles/data/china.osm.pbf
```

---

## 第二步：开始生成

执行：

```bash
make pbf-to-mbtiles area=china
```

---

## Wikidata 已存在

如果之前已经下载过：

```bash
make pbf-to-mbtiles \
area=china \
download=false
```

这样速度更快。

---

## 强制重新下载 Wikidata

```bash
make pbf-to-mbtiles \
area=china \
download=true
```

---

## 指定缩放级别

```bash
make pbf-to-mbtiles \
area=china \
zoom_level=12 \
download=false
```

---

# 7. 区域扩展与地图合并

适用于：

已经有部分地图，后续需要新增区域。

---

## 安装 Osmium

本项目不自带 Osmium 工具，需要单独安装。如，在 Debian / Ubuntu 下执行：

```bash
sudo apt install osmium-tool
```

---

## 例1：

```text
已有：四川地图
新增：重庆地图
最终：川渝地图
```

---

## 合并区域数据

准备：

```text
sichuan.osm.pbf
chongqing.osm.pbf
```

执行：

```bash
osmium merge \
sichuan.osm.pbf \
chongqing.osm.pbf \
-o chuanyu.osm.pbf
```

生成：

```text
chuanyu.osm.pbf
```

---

## 重新生成地图

```bash
make pbf-to-mbtiles area=chuanyu
```

如果 Wikidata 已存在（如，曾为中国地图下载过对应数据）：

```bash
make pbf-to-mbtiles \
area=chuanyu \
download=false
```


---

## 例2：

```text
已有：广东地图
提取：深圳地图
提取：东莞地图
最终：深圳及东莞地图
```

---

## 提取区域

准备：

```text
guangdong.osm.pbf
config.json
```

执行：

```bash
osmium extract \
-c config.json \
guangdong.osm.pbf
```

提取配置示例：

```json
{
  "extracts": [
    {
      "output": "shenzhen.osm.pbf",
      "description": "Shenzhen",
      "bbox": [120.0, 30.0, 121.0, 31.0]
    },
    {
      "output": "dongguan.osm.pbf",
      "description": "Dongguan",
      "polygon": [[
        [9.613465, 53.58071],
        [9.647599, 53.59655],
        [9.649288, 53.61059],
        [9.613465, 53.58071]
      ]]
    }
  ]
}
```

支持：

- 矩形区域提取（Bounding Box）
- 多边形区域提取（Polygon）
- 多个多边形叠加提取（Multipolygon）

---

## 合并区域

```bash
osmium merge \
shenzhen.osm.pbf \
dongguan.osm.pbf \
-o merged.osm.pbf
```

生成：

```text
merged.osm.pbf
```

---

## 重新生成地图

```bash
make pbf-to-mbtiles area=merged
```

如果 Wikidata 已存在（如，曾为广东地图下载过对应数据）：

```bash
make pbf-to-mbtiles \
area=merged \
download=false
```

# 8. 发布离线地图服务

当 MBTiles 生成完成后即可发布。

---

## 检查地图文件

确认文件存在：

```text
client/martin/area.mbtiles
```

---

## 启动服务

执行：

```bash
make publish-mbtiles
```

自动完成：

```text
加载Docker镜像
      ↓
加载Martin配置
      ↓
加载前端配置
      ↓
启动Martin
      ↓
启动Nginx
```

---

# 9. 验证服务是否正常

## 验证 Martin

浏览器访问：

```text
http://localhost:3000/catalog
```

（如部署到服务器，则将localhost替换为服务器地址；下同）

检查：

- 页面可打开
- 返回数据正常
- 能看到 MBTiles 数据集

---

## 验证地图页面

访问：

```text
http://localhost:8088
```

检查：

- 地图正常显示
- 可以缩放
- 可以拖动

---

## 浏览器开发者工具检查

按 F12。

Network 中检查：

- HTTP状态码为200

Console 中检查：

- 无红色错误日志

---

# 10. 停止服务

执行：

```bash
make stop
```

停止：

- Martin
- Nginx
- Docker容器

---

# 11. 推荐操作流程

首次构建：

```text
make all
      ↓
make download-data area=<AREA>
      ↓
make publish-mbtiles
      ↓
访问地图
```

已有PBF：

```text
复制PBF
      ↓
make pbf-to-mbtiles area=<AREA>
      ↓
make publish-mbtiles
      ↓
访问地图
```

新增区域：

```text
Osmium提取
      ↓
Osmium合并
      ↓
生成新的PBF
      ↓
生成MBTiles
      ↓
发布服务
```
