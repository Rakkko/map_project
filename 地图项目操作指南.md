# OpenMapTiles + Martin 离线地图平台操作指南

# 1. 整体流程说明

整个项目由 **Makefile** 统一管理，只需执行对应命令即可完成地图数据构建和地图服务发布。

整体流程如下：

```text
环境初始化
      ↓
获取地图数据（在线下载或本地 PBF）
      ↓
OpenMapTiles 生成 MBTiles
      ↓
复制至 Martin 数据目录
      ↓
启动 Martin + Nginx
      ↓
MapLibre GL JS 显示地图
```

各组件功能如下：

- OSM：原始地图数据
- OpenMapTiles：地图数据处理与矢量瓦片生成
- MBTiles：矢量瓦片数据文件
- Martin：矢量瓦片服务
- Nginx：前端静态资源服务
- MapLibre GL JS：地图显示页面

---

# 2. 项目目录结构

初始化完成后目录如下：

```text
.
├── client
│   ├── docker-compose.yaml
│   ├── docker_image
│   ├── frontend
│   ├── martin
│   │   ├── config.yaml
│   │   └── *.mbtiles
│   └── nginx
│
├── host
│   └── openmaptiles
│
└── Makefile
```

## 目录作用

### client

地图发布环境，负责运行地图服务。

包含：

- Martin 服务
- Nginx 服务
- MapLibre 前端
- Docker 镜像缓存

---

### client/docker_image

存放 Docker 镜像缓存。

初始化后将生成：

```text
martin.tar
nginx.tar
```

离线环境可直接加载，无需重新下载镜像。

---

### client/martin

存放 Martin 配置及 MBTiles。

例如：

```text
client/martin/china.mbtiles
```

Docker Compose 会自动挂载该目录下所有 MBTiles 文件。

---

### host/openmaptiles

OpenMapTiles 工作目录。

负责：

- 下载地图数据
- 导入 OSM 数据
- 生成 MBTiles

---

# 3. 第一次使用：环境初始化

首次使用必须执行一次初始化。

执行：

```bash
make init
```

## 初始化内容

系统自动完成：

```text
检查 Docker
      ↓
检查 Git
      ↓
检查 GNU Make
      ↓
检查 Martin 镜像
      ↓
下载镜像（如需要）
      ↓
保存镜像缓存
      ↓
克隆 OpenMapTiles
      ↓
更新 OpenMapTiles（已存在时）
      ↓
检查 OpenMapTiles 运行环境
```

初始化完成后：

- `host/openmaptiles` 已创建
- `client/docker_image` 已生成镜像缓存
- 工程可直接进行地图构建或发布

# 4. 设置地图最大缩放级别

## 什么是 Zoom Level

Zoom Level 表示地图可放大的最大层级。

例如：

```text
10 → 城市级
14 → 街道级
18 → 建筑级
20 → 高精度显示
```

默认值一般为：

```text
14
```

---

## 修改缩放级别

执行：

```bash
make set-max-zoom zoom_level=20
```


---

## 缩放级别越高有什么影响

| 项目 | 影响 |
|--------|--------|
| 生成时间 | 增加 |
| 磁盘占用 | 增加 |
| MBTiles大小 | 增加 |
| 内存占用 | 增加 |

# 5. 在线下载地图并发布

适用于联网环境。

系统将自动下载指定区域地图、生成 MBTiles，并启动地图服务。

---

## 下载中国地图

```bash
make online-publish area=china
```

---

## 下载四川地图

```bash
make online-publish area=asia/china/sichuan
```

---

## 自动执行内容

系统将依次执行：

```text
检查运行环境
      ↓
启动 OpenMapTiles
      ↓
下载 OSM 数据
      ↓
导入 OSM
      ↓
导入 Wikidata
      ↓
生成 SQL
      ↓
生成 Bounding Box
      ↓
生成 MBTiles
      ↓
复制至 client/martin
      ↓
启动 Martin
      ↓
启动 Nginx
```

最终生成：

```text
client/martin/<AREA>.mbtiles
```

并自动发布地图服务。

# 7. 区域扩展与地图合并

适用于：

已经有部分地图，后续需要新增区域。

---

## 安装 Osmium

本项目不自带 Osmium 工具，需要单独安装。如，在 Debian / Ubuntu 下执行：

```bash
sudo apt install osmium-tool
```

---

## 例1：

```text
已有：四川地图
新增：重庆地图
最终：川渝地图
```

---

## 合并区域数据

准备：

```text
sichuan.osm.pbf
chongqing.osm.pbf
```

执行：

```bash
osmium merge \
sichuan.osm.pbf \
chongqing.osm.pbf \
-o chuanyu.osm.pbf
```

生成：

```text
chuanyu.osm.pbf
```

---

## 重新生成地图

```bash
make pbf-to-mbtiles area=chuanyu
```

如果 Wikidata 已存在（如，曾为中国地图下载过对应数据）：

```bash
make pbf-to-mbtiles \
area=chuanyu \
download=false
```


---

## 例2：

```text
已有：广东地图
提取：深圳地图
提取：东莞地图
最终：深圳及东莞地图
```

---

## 提取区域

准备：

```text
guangdong.osm.pbf
config.json
```

执行：

```bash
osmium extract \
-c config.json \
guangdong.osm.pbf
```

提取配置示例：

```json
{
  "extracts": [
    {
      "output": "shenzhen.osm.pbf",
      "description": "Shenzhen",
      "bbox": [120.0, 30.0, 121.0, 31.0]
    },
    {
      "output": "dongguan.osm.pbf",
      "description": "Dongguan",
      "polygon": [[
        [9.613465, 53.58071],
        [9.647599, 53.59655],
        [9.649288, 53.61059],
        [9.613465, 53.58071]
      ]]
    }
  ]
}
```

支持：

- 矩形区域提取（Bounding Box）
- 多边形区域提取（Polygon）
- 多个多边形叠加提取（Multipolygon）

---

## 合并区域

```bash
osmium merge \
shenzhen.osm.pbf \
dongguan.osm.pbf \
-o merged.osm.pbf
```

生成：

```text
merged.osm.pbf
```

---

## 重新生成地图

```bash
make pbf-to-mbtiles area=merged
```

如果 Wikidata 已存在（如，曾为广东地图下载过对应数据）：

```bash
make pbf-to-mbtiles \
area=merged \
download=false
```

# 8. 发布已有 MBTiles

如果已经拥有 MBTiles 文件，可直接启动地图服务。

---

## 第一步：复制 MBTiles

例如：

```text
china.mbtiles
```

放入：

```text
client/martin/
```

---

## 第二步：发布地图

执行：

```bash
make publish-mbtiles area=china
```

系统将自动：

```text
检查运行环境
      ↓
检查 MBTiles
      ↓
检查 Docker 镜像
      ↓
加载镜像（如需要）
      ↓
启动 Martin
      ↓
启动 Nginx
```

如果服务已运行，则仅重启 Martin，使新增 MBTiles 生效。

# 9. 验证服务是否正常

## 验证 Martin

浏览器访问：

```text
http://<服务器地址>:3000/catalog
```

检查：

- 页面可正常打开
- 返回 Catalog 信息
- 能看到对应 MBTiles 数据集

---

## 验证地图页面

浏览器访问：

```text
http://<服务器地址>:8088
```

检查：

- 地图正常显示
- 可以缩放
- 可以拖动

---

## 浏览器开发者工具

按 **F12** 查看：

### Network

检查：

- 地图请求状态码为 **200**

### Console

检查：

- 无红色错误日志
- 无资源加载失败信息

---

# 10. 停止服务

make stop

- Martin
